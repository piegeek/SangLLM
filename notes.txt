Basically, to start up, we need the following objects:

1. dataset
- Holds tokenIDs, vocab_size, context_length(window size)

2. loader
- Loads (input, output) pairs for training
- Each input & output has dimensions (B x T)

3. model
- Holds embedding matrix

4. optimizer
- Used for backpropagation

Batch size vs Sequence length

Sequence length: window size
Batch size: How many training examples we process simultaneously -> Affects training efficiency

Q. What's the difference between attention.Head.forward() and model.AttentionLM.forward()

A. 
The AttentionLM.forward() is the entire model's forward pass.
The Head.forward() is just one layer inside the model.

x: (B, T, d_model) -> (B, T, head_size) (Head); (B, T, C) -> (B, T, C) if d_model=C and head_size=C in our code
idx: (B, T) -> (B, T, vocab_size) (AttentionLM)

When calling
logits = model(x)

(B, T) -> (B, T, d_model) -> (B, T, vocab_size)

Q. 
What does FeedForward(MLP) do in terms of the overall architecture?

A. 
its job is to refine whatever the attention block gathered. (GELU)
Also, without FFN, tokens only mix information, with FFN, tokens mix + compute new features

[6/11/26]

1. dataset.py -> Generates the dataset that is to be loaded into loader
2. model.py -> Defines the total LLM model. Feeds input embeddings (tok_emb + pos_emb, (B, T, d_model)) into the Transformer blocks and generates a logit (B, T, vocab_size) output
3. attention.py -> Defines SingleHeadAttention, MultiHeadAttention. Maps (B, T, d_model) into (B, T, head_size), then when we get the output, we project it back to (B, T, d_model)
4. tokenizer.py -> Defines various tokenization schemes
5. train.py -> 1. Set tokenizer, create dataset, load from dataset to get idx (B, T)
			   2. Set model, optimizer, then go through indices and check loss and backpropagagte
			   3. Once training is finished, store the model in *.pt file
